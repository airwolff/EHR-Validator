One word, defined

Severity = is this number believable? Not: is the patient in danger. Blood oxygen above 100% is critical because 100 is the maximum — it's impossible data.

The claim I'll back with measurements

When the rule is simple and you already know it, plain code wins — faster, cheaper, same answer every time. The AI is for the jobs code can't do: reading the doctor's notes, catching two things that contradict each other, spotting a pattern nobody wrote a rule for.

What the checker can't do — 1

It won't stop a hidden instruction. Text telling the AI what to do, hiding inside a note, gets quoted word for word — so the checker keeps it.

What the checker can't do — 2

Word-for-word isn't the same as honest. A model can quote "dysuria" out of "denies dysuria". That's why a human works the list.

I did not let a language model decide what runs next over patient data.

Way Forward

Four things I'd build next

A referee deferred, not dropped

When the identity agent and the clinical agent disagree about the same record, nothing resolves it today — both findings just sit there. A third agent judging that disagreement is a real job for an AI; rules can't do it.

A web page

So this is something a person clicks instead of something I type.

The database on a real server

The code is already written to run there without changes. The free tier's 30-day clock is the only catch, and synthetic data reloads fine.

Real notes are messier than mine

Every record here is synthetic and every problem in it was planted by me. Before this touches anything real, the grading and the bias framing both need to be rebuilt.

Close

Every claim here traces back to a saved recording or a SQL query you can run yourself.

The rules kept the jobs with fixed specs. The agents took the jobs that need reading. And a layer that isn't AI at all checked the agents' work before any of it counted.

I gave each job to the tool that's good at it.
